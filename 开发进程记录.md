## 2026-06-11（压花损耗正式计入成本）

### 背景

- 演示中发现设置「压花损耗率(每道)」后计算结果不变。核查确认：该变量自 2026-04-17 引入时即为占位（仅展示损耗面积，口径待定），从未进入成本公式。

### 业务口径（本次确认并冻结）

- 压花的“原材料”是已完成其他工艺的非压花产品，每道压花按比例报废一部分。
- **压花前投产面积 = 基准面积 ÷ (1 - 每道损耗率)^压花道数**（逐道嵌套）。
  - 例：损耗 10%，成品需 100 ㎡：1 道 → 100/90%；2 道 → (100/90%)/90% ≈ 123.457 ㎡。

### 实现变更

- `core/calculator.py`：损耗放大移入步骤1（面积链路），`初步总面积 = 压花前投产面积 / (1 - 不良品率) + 试机面积`。铝材、油漆、前处理、包装、吨基费、压花加工费等按面积/重量计的成本全部联动。
- 新增结果字段：`pre_embossing_area`（压花前投产面积）、`embossing_loss_rate`；`embossing_loss_area` 改由步骤1产出。
- 校验：压花道数 > 0 时，损耗率必须在 [0, 1) 区间，否则拒绝计算。
- `core/reporting.py`：步骤1报告新增「压花前投产面积」「压花损耗面积」两行（仅压花道数 > 0 时显示），「初步总面积」公式同步改写；兼容旧存档（无新字段时回退为基准面积）。
- 界面：变量单位标注由「-」改为「比例 0-1」。

### 验证

- `tests/test_cost_regression.py` 新增 4 个压花损耗回归测试（嵌套公式、成本联动、0 道免疫、非法损耗率拒绝），全部 8 个测试通过。
- 端到端复核：100 ㎡ / 损耗 10% / 2 道 → 压花前投产面积 123.457 ㎡，与业务口径示例一致。

## 2026-04-18（存档与体验类迭代）

### 基线存档

- **本轮存档文件**：`archives/ALUCOLUX_CostCalculator_milestone_20260418_091949.zip`
- **包内主要包含**：`app.py`、`core/`、`requirements.txt`、`color_cost_db.csv`、`default_config.json`、`saved_calculations/`（含本机已保存计算）、`开发进程记录.md`、两份 ALUCOLUX 规格/方法说明 md、`阶段性开发总结_2026-04-16.md`。

### 代码结构维护

- **`app.py` 清理**：删除与 `core` 重复的旧实现块，底部保留对 `core.calculator` / `core.optimizer` / `core.reporting` / `core.storage` 的委托入口，降低同名函数覆盖风险。
- **注释规范**：在 `core/calculator.py`、`core/optimizer.py`、`core/reporting.py`、`core/storage.py` 及 `app.py` 关键入口补充**面向非程序员**的中文业务说明（用途、口径、修改影响），便于业务同事与 AI 辅助维护。

### 显示与输入精度

- **米制长度**：单板宽/长、试机长、卷头尾长等以 **m** 展示时统一 **小数 3 位**（毫米级）；变量区 `TRIAL_LENGTH`、`HEAD_TAIL_LENGTH` 同步为 `%.3f`。
- **面积（㎡）**：报告与优化文案中面积类数值统一 **最多 3 位小数**；**合同面积**在界面输入为 **整数 ㎡**（`step=1`、`%.0f`）；**开机费阈值面积**变量区为 `%.3f`、`step=1`。
- **单板长度默认值**：由 2.44 m 调整为 **3.00 m**。

### 计算库（优化多选）列表标签

- 字段顺序调整为：**项目名 → 合同总规模（㎡，整数）→ 色 → 宽（整数 mm）厚（整数 mm）→ 美元单价（USD/㎡，固定 3 位小数）→ 保存时间**。
- 不再在标签中展示涂层、板长、人民币总成本，以缩短多选行。
- **优化 payload**：`result` 中增加 `usd_price`，新存档可直接展示美元单价；旧存档无该字段时用 `break_even_per_m2 ÷ EXCHANGE_RATE` 推算。

### 项目名称与计算库去重

- **默认项目名**：首次进入时自动生成 `项目{n}` / `Project {n}` 中**第一个与计算库已有名称不冲突**的名称。
- **重名校验**：若当前输入与库中某条 `project_name`（strip 后）**完全相同**，界面警告 + 提供「建议名称」与「填入建议名称」按钮；点击「一键计算」时若仍重复则**阻止计算**并提示。
- **保存到库后**：自动将输入框中的项目名刷新为**新的不重名默认**，便于连续录入。

### 品牌展示

- 用户可见文案中的品牌名统一为 **ALUCOLUX®**（含页面标题、中英文应用标题、成本报告与优化报告标题）。
- **`[ALUCOLUX_OPTIMIZER_DATA]`** 等嵌入标记**不改写**，避免破坏历史导出文件的解析。

### 验证

- `python -m py_compile` 覆盖 `app.py` 与 `core/*.py` 通过。
- IDE 静态检查无新增问题。

### 后续可选

- Streamlit 对 `use_container_width` 的弃用提示：可按官方建议逐步改为 `width='stretch'` 等。
- 下载文件名是否使用 `ALUCOLUX®` 前缀：需评估各操作系统对文件名 Unicode 的兼容性。

## 2026-04-17（优化前封版）

### 本轮新增与修复

- **成本优化模块增强**
  - 新增本地计算库（`saved_calculations`）用于集中管理历史计算。
  - 支持多选历史记录直接运行优化，减少反复上传文件。
  - 优化结果新增单位面积指标：独立/统筹/节约（元/㎡），并提供加权平均口径。
  - 新增同宽厚同色小单合并后的开机费统筹节约逻辑。

- **订单字段扩展**
  - 新增 `项目名称`（用于记录与优化分组展示）。
  - 新增 `分批下单`：按批次重复计入试机与漆盘费。
  - 新增 `压花道数`（0/1/2），压花成本仅在道数 > 0 时生效。

- **压花逻辑修正**
  - 修复“压花成本无条件叠加”问题。
  - 新增变量 `EMBOSSING_LOSS_PER_PASS`（默认 0），当前仅展示损耗面积，不影响成本（口径待定）。

- **颜色库与联动修复**
  - 颜色库新增 `embossing_passes` 字段（CSV/新增记录/维护表格全链路支持）。
  - 修复“选中颜色后涂层类型带入不稳定”问题。
  - 选色后现可稳定同步：涂层类型 + 压花道数 + 面漆价 + 清漆价。

- **稳定性修复**
  - 修复优化模块 `KeyError: 'BAD_RATE'`：优化 payload 改为完整变量快照，并对旧记录自动补齐缺失变量。

### 当前状态

- 主流程计算、导出、颜色库维护、计算库存档、优化分析均可运行。
- 已通过语法检查与静态检查，可作为“优化前基线版本”。

### 已完成变更

- **铝价体系收敛**
  - 删除 SMM 相关配置、选择逻辑、报告展示与计算分支。
  - 计算统一使用 `AL_PRICE_A00_CHANGJIANG` 同步到 `AL_PRICE`。

- **长江查询应用修复**
  - “应用到计算”时，查询均价（元/吨）先换算为元/kg再写入计算值。
  - 增加输入控件版本号刷新机制，避免 Streamlit 旧控件状态覆盖新值。

- **铝成本口径调整**
  - 新增并启用 `AL_COIL_PROCESSING_FEE`（默认 `2.2`，单位元/kg）。
  - 铝材成本改为：
    - `铝材成本 = 总铝重 ×（铝锭价 + 铝卷加工费）`
  - 报告中同步展示锭价成本、铝卷加工费成本与合计。

- **命名与文案统一**
  - 界面“长江A00（元/kg）”改为“铝锭价（元/kg）”。
  - “铝加工费”统一为“铝卷加工费”。
  - 中文报告中的关键变量说明改为“中文名 + 【变量名】”风格，降低中英混排。

- **单位规范化**
  - `AL_DENSITY` 显示单位由 `kg/m²/mm` 规范为 `kg/(㎡·mm)`。

- **兼容性处理**
  - 导入旧配置时，自动兼容旧键 `AL_PROCESSING_FEE_PER_M2` 到新键 `AL_COIL_PROCESSING_FEE`。
  - 运行中如存在旧键会自动迁移并清理。

### 当前计算口径（冻结）

- 铝锭价：元/kg（由用户输入值决定；网络查询仅作参考，点击应用后才写入）。
- 铝卷加工费：元/kg（默认 2.2）。
- 铝材成本：总铝重 ×（铝锭价 + 铝卷加工费）。
- 铝密度：`kg/(㎡·mm)`，默认值 2.73。
