# ALUCOLUX 成本计算器阶段性开发总结（存档）

日期：2026-04-16  
版本：v0.2.0  
项目：ALUCOLUX Cost Calculator

## 一、阶段目标回顾

本阶段目标是先完成一个可运行、可验证、规则清晰的成本计算器原型，重点保证：

- 计算逻辑严格变量驱动（避免写死）
- 核心业务规则可配置、可复用
- 报告输出可直接用于报价沟通
- UI 可在中英文间切换，支持后续国际化

## 二、已完成功能

### 1) 应用框架与运行

- 已完成 `Python + Streamlit` 应用搭建
- 程序可本地运行并通过浏览器访问
- 已补充依赖文件：`requirements.txt`

### 2) 订单输入与业务规则

已支持订单核心输入：

- 合同面积
- 单板宽度
- 单板长度
- 板厚
- 涂层类型（必选）
- 试机次数（默认值 + 手工覆盖）

涂层规则已固化：

- `PVDF2`：底漆 + 面漆（无清漆）
- `PVDF3`：底漆 + 面漆 + 清漆
- `1花`：底漆 + 面漆 + 印花1层 + 清漆
- `2花`：底漆 + 面漆 + 印花2层 + 清漆

### 3) 计算引擎

已完成 5 步完整计算：

1. 最终总生产面积
2. 铝材成本
3. 油漆成本（含面漆漆盘费）
4. 其他直接成本
5. 总成本与单价

已实现并验证的重要规则：

- 合同面积独立输入（不由宽长反推）
- 开机费按合同面积阈值判断
- 漆盘费仅对面漆生效
- 压花成本变量 `EMBOSSING_PRICE` 已加入（默认 0 元/㎡）

### 4) 高级损耗模型

已新增可开关模型：

- 长度整除浪费模型（按单板面积向上取整块数）
- 开启后使用取整后的面积进入损耗计算
- 报告中显示取整相关过程与浪费面积

### 5) 配置管理

已支持：

- 导入配置（JSON）
- 导出配置（JSON）
- 恢复默认变量（恢复系统出厂值）
- 保存为默认参数（保存到本地 `default_config.json`，下次启动自动加载）

### 6) 报告与导出

支持导出格式：

- TXT
- Markdown
- RTF

报告内容已包含：

- 版本号
- 参数快照 ID
- 生成时间
- 订单参数与 5 步计算明细

### 7) 国际化（i18n）阶段成果

已支持中英界面切换（侧边栏选择）：

- 订单输入区文案
- 计算输出区文案
- 报告标题与步骤标题
- 英文报告已清理中文残留

变量区标签已优化为：

- 友好名称 + 单位（不再展示变量代号）

### 8) UI/UX 优化

- 变量参数区改为默认折叠，减少主界面干扰
- 默认输入值优化：
  - 宽度默认 `1.5 m`
  - 厚度默认 `3.0 mm`

## 三、当前可用状态

当前版本已可用于：

- 日常打样/报价的快速测算
- 不同参数体系的对比测算
- 中英文场景下的演示与沟通

并具备基础存档能力（参数快照 + 配置导入导出）。

## 四、已确认的关键业务口径（冻结）

- 合同面积是独立给定值
- 单板宽长用于制造规格与整除浪费模型
- 涂层类型为必填输入
- 输出采用 5 步结构（不再强制第 6 步）
- 输出格式允许非 Markdown（TXT / RTF 均可）

## 五、后续建议（下一阶段）

建议下一阶段按优先级推进：

1. **全量国际化**
   - 把报告正文每条公式说明进一步结构化为语言模板，彻底避免维护分叉
2. **参数版本管理**
   - 增加“参数方案名称”和“历史方案列表”
3. **数据校验增强**
   - 对异常输入（极端厚度、极高不良率等）提供业务提示
4. **打包交付**
   - 增加 Windows 启动器与一键构建脚本，形成可分发版本
5. **测试用例固化**
   - 建立 4 种涂层的回归样例，避免后续迭代引入计算回归
