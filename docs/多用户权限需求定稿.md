# ALUCOLUX® 多用户权限需求定稿

> **文档状态**：已实现（app v0.3.0）  
> **版本**：v1.1  
> **记录日期**：2026-06-11  
> **适用范围**：网络部署场景下的 Streamlit 版本（v0.2.1 及后续）

---

## 1. 背景与目标

将成本计算器从「单机/便携、人人同等权限」扩展为**多用户网络部署**，通过独立账号与三级角色，实现：

- 计算参数与系统配置由管理员统一维护；
- 颜色库按角色分级维护，CSV 批量操作仅管理员；
- 计算库按用户隔离，高级用户仅操作自己的记录，管理员可查看全部；
- 普通用户可完成报价所需输入与计算，但**不暴露运算过程与分项成本**，仅展示最终报价数字。

---

## 2. 角色定义

| 角色 | 内部代码建议 | 说明 |
|------|--------------|------|
| 管理员 | `admin` | 全部权限；可管理账号、重置密码 |
| 高级用户 | `advanced` | 完整运算报告与导出；可维护颜色（增改不可删）；可使用计算库与优化（仅自己的记录） |
| 普通用户 | `basic` | 订单输入、选色、计算；结果仅显示总价与单价（含美元）；不可维护颜色库与计算库 |

---

## 3. 账号与认证

| 项 | 定稿 |
|----|------|
| 登录方式 | 每人**独立账号**（用户名 + 密码） |
| 密码管理 | **管理员可重置**任意用户密码 |
| 会话 | 登录后绑定 `username` 与 `role`；未登录不可使用主功能 |
| 账号存储 | 服务器本地配置文件（建议 `数据/users.json`），密码仅存哈希，**不提交 git** |
| 安全边界 | 应用层权限，适用于内网小团队；界面隐藏 + 写盘前校验双重防护 |

### 3.1 v1 暂不强制项（可后续迭代）

- 管理员重置密码后，**不强制**用户下次登录必须改密；
- 操作审计日志（谁何时改了默认参数/颜色库）留作后续可选功能。

## 4. 功能权限矩阵

图例：✅ 允许　❌ 禁止　🔒 仅自己的数据　🌐 全部数据

### 4.1 订单与计算（全员基础能力）

| 功能 | 管理员 | 高级 | 普通 |
|------|:------:|:----:|:----:|
| 订单输入（项目名、合同面积、宽长厚等） | ✅ | ✅ | ✅ |
| 从颜色库选色（下拉/搜索） | ✅ | ✅ | ✅ |
| 手动改涂层 / 压花 / 试机次数等 | ✅ | ✅ | ✅ |
| 铝锭价：长江查询 + 手动修改 | ✅ | ✅ | ✅ |
| 一键计算 | ✅ | ✅ | ✅ |
| 界面语言切换 | ✅ | ✅ | ✅ |

> 铝锭价会参与当次计算，但**不通过「变量参数」区修改**；高级/普通用户无变量参数区入口。

---

### 4.2 计算参数与配置

| 功能 | 管理员 | 高级 | 普通 |
|------|:------:|:----:|:----:|
| 变量参数区（侧栏 + 主区 expander） | ✅ 可编辑 | ❌ 不显示 | ❌ 不显示 |
| 配置 JSON 导入 | ✅ | ❌ | ❌ |
| 配置 JSON 导出 | ✅ | ❌ | ❌ |
| **保存为默认参数** | ✅ | ❌ | ❌ |
| **恢复已保存的默认参数** | ✅ | ✅ | ✅ |

> 「恢复默认」：从服务器 `default_config.json` 载入管理员保存过的全局默认；全员可恢复，仅管理员可写入默认文件。

---

### 4.3 计算结果与报告

| 功能 | 管理员 | 高级 | 普通 |
|------|:------:|:----:|:----:|
| 完整运算过程（步骤 1–5） | ✅ | ✅ | ❌ |
| 分项成本明细（铝材、油漆、试机等各项金额） | ✅ | ✅ | ❌ |
| **总价**（总直接成本，元） | ✅ | ✅ | ✅ |
| **平米单价**（元/㎡） | ✅ | ✅ | ✅ |
| **美元单价**（USD/㎡） | ✅ | ✅ | ✅ |
| 投产面积、铝重等物理量 | ✅（在完整报告中） | ✅ | ❌ |
| 导出报告（TXT / Markdown / RTF） | ✅ | ✅ | ❌ |

#### 普通用户计算结果展示（最终定稿）

计算完成后，**仅显示以下 3 项**，以摘要卡片形式呈现：

1. **总价**（元）
2. **平米单价**（元/㎡）
3. **美元单价**（USD/㎡）

**不得展示**：运算步骤、分项成本、物理量、导出按钮及任何其他字段。

#### 管理员 / 高级用户

- 展示完整运算过程报告；
- 可导出 TXT / Markdown / RTF。

---

### 4.4 颜色库

| 功能 | 管理员 | 高级 | 普通 |
|------|:------:|:----:|:----:|
| 选色使用（订单区） | ✅ | ✅ | ✅ |
| 新增 / 更新单条颜色记录 | ✅ | ✅ | ❌ |
| 表格维护保存（主区 data_editor） | ✅ | ✅（可增改，**不可删行**） | ❌ |
| 删除颜色记录 | ✅ | ❌ | ❌ |
| 颜色库 **CSV 导入** | ✅ | ❌ | ❌ |
| 颜色库 **CSV 导出** | ✅ | ❌ | ❌ |

> **CSV 级操作（导入、导出、批量替换）仅管理员。**  
> 高级用户仅能通过界面单条新增或表格增改；保存时不得减少记录总数（不可删除）。

---

### 4.5 计算库与成本优化

| 功能 | 管理员 | 高级 | 普通 |
|------|:------:|:----:|:----:|
| 保存当前计算到库 | ✅ | ✅ | ❌ |
| 查看计算库列表 | 🌐 全部 | 🔒 仅自己 | ❌ |
| 多选并运行统筹优化 | 🌐 可对全部 | 🔒 仅自己的记录 | ❌ |
| 删除计算库记录 | 🌐 可删全部 | 🔒 仅删自己的 | ❌ |
| 从文件导入优化（备选入口） | ✅ | ✅ | ❌ |
| 优化报告查看与下载 | ✅ | ✅ | ❌ |

#### 计算库数据隔离规则

- 每条保存记录增加字段 **`owner_username`**（保存时写入当前登录用户）；
- 高级用户：列表、多选、优化、删除均**仅针对本人记录**；
- 管理员：可见并操作全部记录；
- 普通用户：不使用计算库与优化模块。

## 5. 需求确认问答（记录）

| # | 问题 | 定稿结论 |
|---|------|----------|
| 1 | 普通用户算完后看什么？ | **仅摘要：总价 + 平米单价（元/㎡）+ 美元单价（USD/㎡）**；无过程、无分项、无物理量 |
| 2 | 普通用户能否改涂层/压花/试机？ | **可以** |
| 3 | 高级用户能否 CSV 导入颜色库？ | **不可以**（CSV 导入/导出仅管理员） |
| 4 | 高级用户能否用计算库与优化？ | **可以**（仅自己的记录；可保存、查看、优化、删除自己的） |
| 5 | 登录方式？ | **每人独立账号；管理员可重置密码** |

### 5.1 口径修正记录

- **2026-06-11 初稿**：曾将「不要成本」理解为普通用户不显示任何金额。  
- **2026-06-11 更正**：「不要成本」指**不要成本明细与运算过程**；普通用户**必须看到**总价、平米单价及美元单价。

## 7. 界面结构预期（概念）

```
登录页
  └─ 主界面
       ├─ 侧栏
       │    ├─ 语言（全员）
       │    ├─ [管理员] 配置管理（JSON 导入/导出、保存为默认）
       │    ├─ [全员] 恢复已保存的默认参数
       │    └─ [管理员] 颜色库 CSV 导入/导出
       ├─ 订单输入（全员）
       ├─ 铝锭价专区（全员：查询 + 手动修改）
       ├─ [仅管理员] 变量参数 expander
       ├─ [管理员 / 高级] 颜色维护区（权限不同：高级不可删）
       ├─ 一键计算（全员）
       ├─ 计算输出
       │    ├─ [管理员 / 高级] 完整报告 + 导出
       │    └─ [普通] 摘要卡片：总价、元/㎡、USD/㎡
       └─ [管理员 / 高级] 计算库 + 优化（高级仅自己的记录）
```

## 8. 建议实现顺序（待开发阶段执行）

1. 账号模块：`users.json`、登录页、管理员重置密码

4. 计算库隔离：`owner_username` + 列表过滤  
5. 普通用户摘要：`build_report_summary_quote()`（仅 3 个报价数字）  
6. 三角色联调与场景测试
